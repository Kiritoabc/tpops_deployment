# 功能设计文档（Plan）

本目录存放 **实现前** 的设计说明，约定如下：

1. **新功能先写 Plan，再写代码**  
   新建 `plan/plan-<功能简述>.md`（英文小写 + 连字符，便于排序与检索），评审通过后再开发。

2. **Plan 建议包含**  
   - 背景与目标  
   - 范围与非目标  
   - 领域模型 / 数据约定  
   - API 与权限（如适用）  
   - 与现有模块的衔接（如 `runner`、WebSocket）  
   - 分期（MVP / 二期）  
   - 待产品确认的问题  

3. **实现完成后**  
   可在同一文件末尾追加「实现记录」小节，或仅在 `README.md` / CHANGELOG 中写一句入口链接回本文档。
